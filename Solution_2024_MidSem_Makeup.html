Mean & Five-Number Summary — Descriptive Statistics

⭐ 3 Marks Descriptive Statistics Skewness
📖 Theory Background

Descriptive Statistics — Understanding Your Data at a Glance

Descriptive statistics summarise the key features of a dataset without looking at every value. Two of the most powerful tools are the mean and the five-number summary.

  • Mean (x̄): The arithmetic average — sum all values and divide by n. It is pulled toward outliers.
  • Median (Q2): The middle value when data is sorted. Robust to outliers. For odd n, it is the ((n+1)/2)-th value.
  • Quartiles: Q1 splits the lower 25%; Q3 splits the upper 25%. IQR = Q3 − Q1.
  • Five-Number Summary: {Min, Q1, Median, Q3, Max} — gives a complete picture of spread and center.
  • Symmetry test: Mean ≈ Median → symmetric | Mean > Median → right (positive) skew | Mean < Median → left (negative) skew.
\[\bar{x} = \frac{\sum_{i=1}^{n} x_i}{n} \qquad \text{Five-Number Summary: } \{Min,\; Q_1,\; Q_2,\; Q_3,\; Max\}\]

Skewness rule: Mean > Median → positive skew  |  Mean < Median → negative skew

⚙️ Step-by-Step Algorithm
1
Sort the data ascending
WHY: All percentile-based calculations require ordered data
HOW: Write values from smallest to largest
2
Calculate Mean = (Σxᵢ)/n
WHY: Provides the center-of-mass for the dataset
HOW: Add every value, divide by total count n
3
Find Median (Q2)
WHY: Middle value divides data into equal halves
HOW: Odd n → ((n+1)/2)-th value | Even n → average of n/2 and n/2+1 values
4
Find Q1 and Q3
WHY: Quartiles describe the spread of the middle 50%
HOW: Q1 = median of lower half; Q3 = median of upper half (excluding median)
5
Compare Mean vs. Median to assess symmetry
WHY: Their relationship reveals the direction of skewness
HOW: Mean > Median → right-skewed | Mean < Median → left-skewed | Mean ≈ Median → symmetric

Sampling Distribution — Central Limit Theorem

⭐ 2 Marks Central Limit Theorem Z-Score
📖 Theory Background

The Central Limit Theorem (CLT)

The CLT states that if we take many random samples of size n from any population with mean µ and standard deviation σ, then the distribution of sample means \(\bar{X}\) approaches a Normal distribution as n increases — regardless of the population's shape!

  • Standard Error (SE): The std dev of the sampling distribution: SE = σ/√n. A larger sample → smaller SE → more precise estimate.
  • Z-score: Converts a sample mean to a standard normal value so we can use the Z-table.
  • Symmetry: P(Z < −z) = P(Z > z) for the standard normal distribution.
  • Rule of thumb: n ≥ 30 for CLT to apply (though n=49 is excellent).
\[\bar{X} \sim N\!\left(\mu,\; \frac{\sigma^2}{n}\right) \qquad SE = \frac{\sigma}{\sqrt{n}} \qquad Z = \frac{\bar{X} - \mu}{SE}\]
⚙️ Algorithm: CLT Probability Problems
1
Identify µ, σ, and n from the problem
WHY: These three values completely define the sampling distribution
HOW: Read directly from problem statement
2
Compute SE = σ / √n
WHY: SE is the standard deviation of the sampling distribution
HOW: Divide σ by the square root of the sample size
3
Convert each boundary to Z = (x̄ − µ)/SE
WHY: Standardization lets us use the Z-table
HOW: Subtract µ, divide by SE for each boundary value
4
Use Z-table: P(z₁ < Z < z₂) = Φ(z₂) − Φ(z₁)
WHY: Φ gives the area under the standard normal curve
HOW: Look up z₂ and z₁ in Z-table, subtract
🧪 Practice Examples
✏️ Practice 1 — The 68% Rule

µ=100, σ=15, n=25. Find P(97 < X̄ < 103).

SE = 15/5 = 3. Z₁ = (97−100)/3 = −1. Z₂ = (103−100)/3 = +1.

P(−1 < Z < 1) = 0.8413 − 0.1587 = 0.6827 (≈ 68%)

✏️ Practice 2 — One-Tailed

µ=50, σ=10, n=100. Find P(X̄ > 52).

SE = 10/10 = 1. Z = (52−50)/1 = 2. P(Z > 2) = 1 − 0.9772 = 0.0228

✏️ Practice 3 — Wider Interval

µ=200, σ=20, n=64. Find P(196 < X̄ < 204).

SE = 20/8 = 2.5. Z₁ = −1.6, Z₂ = 1.6. P(−1.6 < Z < 1.6) = 0.9452 − 0.0548 = 0.8904

🎯 Actual Exam Question Solution

Naive Bayes Classifier — Spam Detection

⭐ 5 Marks Naive Bayes Laplace Smoothing Machine Learning
📖 Theory Background

Naive Bayes Text Classifier

Naive Bayes is a probabilistic classifier based on Bayes' Theorem. It's called "naive" because it assumes all words (features) are conditionally independent given the class. Surprisingly, this simplification works very well in text classification.

  • Prior P(Class): Proportion of training messages belonging to each class.
  • Likelihood P(word|Class): How often a word appears in messages of that class.
  • Laplace Smoothing (α=1): Add 1 to every word count to avoid zero probability for unseen words. Denominator becomes (total words in class + vocabulary size).
  • Classification: Compute score for each class. Pick the class with the higher score.
  • Score: P(Class) × Π P(wordᵢ|Class) for all words in test message.
\[P(\text{Class}|\text{msg}) \propto P(\text{Class}) \prod_{i} P(w_i | \text{Class})\]

With Laplace Smoothing: \(\quad P(w|\text{Class}) = \dfrac{\text{count}(w,\,\text{Class}) + 1}{\text{total words in Class} + |V|}\)

⚙️ Algorithm: Naive Bayes Classification
1
Compute Prior Probabilities P(Class)
WHY: Reflects how common each class is in training data
HOW: P(Spam) = #spam msgs / total msgs; same for Ham
2
Build the Vocabulary V
WHY: Defines all possible features
HOW: List every unique word across all training messages
3
Count words in each class, apply Laplace smoothing
WHY: Smoothing prevents zero-probability for unseen words
HOW: P(w|C) = (count(w,C)+1) / (totalWordsC + |V|)
4
Score each class: Prior × product of likelihoods
WHY: The score represents the (unnormalized) posterior probability
HOW: Multiply the class prior by P(w₁|C)×P(w₂|C)×P(w₃|C)...
5
Assign class with the highest score
WHY: Maximum a posteriori decision rule
HOW: Compare scores; the higher one wins
🧪 Practice Examples
✏️ Practice 1 — Mini Sentiment Analysis

Training: "Good product" (Pos), "Bad quality" (Neg), "Very good" (Pos)

V: {good, product, bad, quality, very} → |V|=5. P(Pos)=2/3, P(Neg)=1/3

Test: "good"  Pos words: good(2),product(1),very(1)=4. Neg: bad(1),quality(1)=2

P(good|Pos)=(2+1)/(4+5)=3/9. P(good|Neg)=(0+1)/(2+5)=1/7

Score Pos=(2/3)(3/9)=6/27≈0.222. Score Neg=(1/3)(1/7)=1/21≈0.048. → POSITIVE ✓

✏️ Practice 2 — The Zero-Probability Problem

Without Laplace smoothing, if "free" never appears in Ham training data: P(free|Ham) = 0.

Then Score(Ham) = 0 for ANY message containing "free" — even if it's clearly ham!

With Laplace (α=1): P(free|Ham) = 1/(total_ham_words + |V|) > 0. Problem solved! ✅

✏️ Practice 3 — Why Use Log-Probabilities

Multiplying many small probabilities (0.01 × 0.02 × 0.003...) creates numerical underflow (rounds to 0).

Solution: log Score = log P(Class) + Σ log P(wᵢ|Class). Logs turn products into sums — no underflow!

The classification result is identical — just use the class with the higher log-score.

Sample Size Determination for a Given Margin of Error

⭐ 2 Marks Confidence Intervals Sample Size
📖 Theory Background

How Large Should Our Sample Be?

A margin of error (E) quantifies the precision of a confidence interval. If we want E to be at most a specific value, we solve for the minimum required sample size n.

  • Larger n → smaller E: More data gives more precision.
  • Higher confidence → larger n: e.g., 99% CI needs bigger sample than 95% CI.
  • z-values: 90% → 1.645 | 95% → 1.960 | 99% → 2.576
  • Always round UP: If n = 208.1, you need 209 — rounding down would make E slightly larger than desired.
\[E = z^* \cdot \frac{\sigma}{\sqrt{n}} \quad \Longrightarrow \quad n = \left(\frac{z^* \cdot \sigma}{E}\right)^2\]

Always round n UP to the nearest integer (ceiling function)

⚙️ Algorithm
1
Determine z* from the confidence level
WHY: Different confidence levels require different critical values
HOW: 90%→1.645 | 95%→1.960 | 99%→2.576
2
Compute n = (z*·σ/E)²
WHY: Algebraic rearrangement of the margin of error formula
HOW: Square the ratio (z*·σ) / E
3
Round UP to nearest whole number
WHY: n must be integer; floor would violate the margin requirement
HOW: n = ⌈calculated value⌉
🧪 Practice Examples
✏️ Practice 1 — 95% CI

σ=10, E=2, 95% → z=1.96. n = (1.96×10/2)² = 9.8² = 96.04 → n = 97

✏️ Practice 2 — 90% CI

σ=25, E=5, 90% → z=1.645. n = (1.645×25/5)² = 8.225² = 67.65 → n = 68

✏️ Practice 3 — Effect of Confidence Level

σ=20, E=3. At 95%: n=(1.96×20/3)²=170.7→171. At 99%: n=(2.576×20/3)²=294.9→295.

Higher confidence nearly doubles the required sample size!

🎯 Actual Exam Question Solution

Given: s = 14 min (use as σ estimate), E = 2.5 min, 99% confidence.

1
Critical value for 99% confidence
\(z^* = 2.576\)
2
Apply sample size formula
\[n = \left(\frac{z^* \cdot \sigma}{E}\right)^2 = \left(\frac{2.576 \times 14}{2.5}\right)^2 = \left(\frac{36.064}{2.5}\right)^2 = (14.4256)^2 \approx 208.10\]
3
Round up
208.10 → n = 209 workers
Final Answer Minimum required sample size = n = 209 workers

Discrete Random Variable — PMF, E[X], Var(X)

⭐ 3 Marks Probability Mass Function Expectation Variance
📖 Theory Background

Probability Mass Function — The Language of Discrete Random Variables

A Probability Mass Function (PMF) P(X = x) assigns a probability to each possible value x of a discrete RV X. For a valid PMF:

  • Non-negativity: P(X=x) ≥ 0 for all x
  • Normalization: Σ P(X=x) = 1 over all possible values
  • Expected Value E[X]: The probability-weighted average — Σ x·P(x). Think of it as the long-run average.
  • E[X²]: The probability-weighted average of x² — Σ x²·P(x). Needed for variance.
  • Variance Var(X) = E[X²] − (E[X])²: Measures spread. Always ≥ 0.
  • Symmetry shortcut: If PMF is symmetric around 0, then E[X] = 0 automatically.
\[\sum_x P(X=x)=1 \qquad E[X]=\sum_x x\,P(x) \qquad \text{Var}(X)=E[X^2]-(E[X])^2\]
⚙️ Algorithm: PMF Problems
1
Find k: set ΣP(x) = 1 and solve
WHY: Normalization is the fundamental property of any PMF
HOW: Substitute each x, sum the expressions, set equal to 1
2
Compute E[X] = Σ x·P(x)
WHY: Gives the long-run average value of X
HOW: Multiply each x by P(x), then add all terms
3
Compute E[X²] = Σ x²·P(x)
WHY: Required for the variance shortcut formula
HOW: Replace x with x² in the expectation computation
4
Var(X) = E[X²] − [E[X]]²
WHY: The computational formula for variance
HOW: Subtract square of mean from E[X²]
🧪 Practice Examples
✏️ Practice 1 — P(X=x) = k(x+1), x=0,1,2

ΣP = k(1+2+3) = 6k = 1 → k = 1/6

E[X] = 0(1/6) + 1(2/6) + 2(3/6) = 8/6 = 4/3 ≈ 1.33

E[X²] = 0(1/6) + 1(2/6) + 4(3/6) = 14/6. Var = 14/6 − (4/3)² = 10/18 ≈ 0.556

✏️ Practice 2 — Discrete Uniform: P(X=x)=k, x=1,2,3,4,5

5k=1 → k=1/5. E[X]=(1+2+3+4+5)/5=3. E[X²]=(1+4+9+16+25)/5=11. Var=11−9=2

✏️ Practice 3 — Symmetric: P(X=x)=kx², x=−2,−1,0,1,2

Bayes' Theorem — The DNA Crime Scene Problem

⭐ 5 Marks Bayes' Theorem Conditional Probability Law of Total Probability
📖 Theory Background

Bayes' Theorem — Rational Reasoning Under Uncertainty

Bayes' Theorem shows how to update a prior belief (what we thought before) into a posterior belief (what we think after seeing evidence). It is one of the most powerful tools in probability and statistics.

  • Prior P(H): Probability of a hypothesis before seeing any evidence.
  • Likelihood P(E|H): How probable the evidence is, assuming the hypothesis is true.
  • Law of Total Probability: P(E) = P(E|H)P(H) + P(E|¬H)P(¬H) — the denominator in Bayes.
  • Posterior P(H|E): Updated probability after seeing the evidence. This is what we seek!

The Prosecutor's Fallacy: Confusing P(DNA match | innocent) with P(innocent | DNA match). These are VERY different! In a large population, several innocent people may also match the DNA profile.

\[P(G|M) = \frac{P(M|G)\cdot P(G)}{P(M)} \qquad P(M)=P(M|G)P(G)+P(M|\bar{G})P(\bar{G})\]
⚙️ Algorithm: Bayes' Theorem
1
Define events G and M clearly
WHY: Precise definitions prevent the Prosecutor's Fallacy
HOW: G = "guilty", M = "DNA matches"
2
Set up all given probabilities
WHY: We need prior, likelihoods, and complement
HOW: P(G)=1/population, P(M|G)=1, P(M|¬G)=false match rate
3
Compute P(M) via Law of Total Probability
WHY: P(M) is the normalizing constant in Bayes' formula
HOW: P(M) = P(M|G)·P(G) + P(M|¬G)·P(¬G)
4
Apply Bayes: P(G|M) = P(M|G)·P(G)/P(M)
WHY: This is the posterior probability of guilt given the match
HOW: Numerator / denominator from step 3
🧪 Practice Examples
✏️ Practice 1 — Medical Test (Base Rate Neglect)

Disease prevalence P(D)=0.01. Sensitivity P(+|D)=0.99. False positive rate P(+|¬D)=0.05.

P(+) = 0.99×0.01 + 0.05×0.99 = 0.0099 + 0.0495 = 0.0594

P(D|+) = 0.0099/0.0594 ≈ 16.7%. Most positive tests are false positives!

✏️ Practice 2 — Spam Filter Update

P(Spam)=0.3. P("deal"|Spam)=0.8. P("deal"|Ham)=0.1.

P("deal") = 0.8×0.3 + 0.1×0.7 = 0.31. P(Spam|"deal") = 0.24/0.31 ≈ 77.4%

✏️ Practice 3 — Factory Quality Control

Machine A makes 60% of parts (P(defect|A)=2%). Machine B makes 40% (P(defect|B)=5%).

P(defect) = 0.012+0.020 = 0.032. P(B|defect) = 0.020/0.032 ≈ 62.5% → Likely from B!

🎯 Actual Exam Question Solution — DNA Evidence

Given: Population = 5,000,000. P(random match) = 1/1,000,000.

1
Define Events & Set Up
G = "suspect is guilty"  |  M = "DNA matches"

\[P(G) = \frac{1}{5{,}000{,}000} \qquad P(\bar{G}) = \frac{4{,}999{,}999}{5{,}000{,}000}\] \[P(M|G) = 1 \quad \text{(perfect match if guilty)}\] \[P(M|\bar{G}) = \frac{1}{1{,}000{,}000} \quad \text{(false match rate)}\]
2
Intuitive Approach — Expected Matches
In 5,000,000 people, the expected number of innocent DNA matches:
\[\text{False matches} = 5{,}000{,}000 \times \frac{1}{1{,}000{,}000} = 5 \text{ people}\] Plus the 1 actual guilty person → total of 6 people match the DNA profile.
3
Apply Bayes' Theorem
\[P(M) = 1\cdot\frac{1}{5\text{M}} + \frac{1}{1\text{M}}\cdot\frac{4{,}999{,}999}{5\text{M}} \approx \frac{1}{5\text{M}} + \frac{5}{5\text{M}} = \frac{6}{5\text{M}}\] \[P(G|M) = \frac{P(M|G)\cdot P(G)}{P(M)} = \frac{1 \cdot \frac{1}{5\text{M}}}{\frac{6}{5\text{M}}} = \frac{1}{6} \approx \mathbf{16.7\%}\]
⚠️
The Prosecutor's Fallacy — A Critical Warning
A DNA match with probability 1 in 1,000,000 does NOT mean there is a 1 in 1,000,000 chance of innocence! In a city of 5 million, we expect 5 innocent people to match. The posterior probability of guilt is only ~16.7% from DNA alone. Additional evidence is absolutely essential!
Final Answer P(Guilty | DNA Match) = 1/6 ≈ 16.7%

Poisson Approximation to Binomial — Machine Breakdowns

⭐ 3 Marks Poisson Distribution Binomial Approximation
📖 Theory Background

Poisson Approximation to Binomial

The Binomial B(n, p) describes n independent trials each with success probability p. When n is large and p is small (rare events), the Poisson approximation simplifies computation dramatically.

  • Condition: n large (≥20) and p small (≤0.05), so that λ = np is moderate.
  • Poisson PMF: P(X=k) = e−λλk/k! — only one parameter: λ.
  • Complement rule: P(X ≥ k) = 1 − P(X ≤ k−1) — much easier than summing infinite terms.
  • Range probability: P(a ≤ X ≤ b) = sum of P(X=k) for k from a to b.
  • Key constant: e−2 ≈ 0.1353. Memorize for λ=2 problems!
\[X \sim \text{Poisson}(\lambda),\quad \lambda = np \qquad P(X=k) = \frac{e^{-\lambda}\,\lambda^k}{k!}\]

\(e^{-1}\approx 0.3679,\quad e^{-2}\approx 0.1353,\quad e^{-3}\approx 0.0498,\quad e^{-3.6}\approx 0.0273\)

⚙️ Algorithm: Poisson Problems
1
Compute λ = np
WHY: λ parameterizes the entire Poisson distribution
HOW: Multiply n by p from the problem
2
For exact k: P(X=k) = e^(−λ) × λ^k / k!
WHY: Direct application of PMF formula
HOW: Compute e^(−λ), multiply by λ^k, divide by k factorial
3
For "at least k": use 1 − P(X ≤ k−1)
WHY: Complement avoids summing an infinite series
HOW: Sum P(X=0) through P(X=k−1), subtract from 1

WHY: No simpler closed form exists for bounded ranges
HOW: Build table of P(X=k) for k=a,a+1,...,b and sum

Poisson Process — Website Visit Probability

⭐ 2 Marks Poisson Process Rate Scaling
📖 Theory Background

Poisson Process — Events Over a Time Window

A Poisson process models random events occurring at a constant average rate λ per unit time. The key property is rate scaling: in t time units, the number of events follows Poisson(λt).

  • If a website gets λ=1.2 visits/hour, then in t=3 hours: X ~ Poisson(1.2×3 = 3.6)
  • Events in non-overlapping intervals are independent of each other.
  • P(X ≥ 2) = 1 − P(X=0) − P(X=1) = 1 − e−λt − λt·e−λt = 1 − e−λt(1 + λt)
\[\text{In } t \text{ time units: } X \sim \text{Poisson}(\lambda t)\] \[P(X \ge 2) = 1 - e^{-\lambda t}(1 + \lambda t)\]

⭐ 5 Marks Continuous RV PDF Integration Expected Value
📖 Theory Background

Continuous Random Variables & the PDF

A continuous random variable X can take any value in a range. Its distribution is described by a Probability Density Function (PDF) f(x), where probability = area under the curve.

  • Non-negativity: f(x) ≥ 0 everywhere
  • Normalization: ∫−∞∞ f(x)dx = 1 (total area must be 1)
  • Probability: P(a < X < b) = ∫ab f(x)dx
  • Expected Value: E[X] = ∫ x·f(x)dx (weighted center of mass)
  • Piecewise PDF: Different formula on different intervals — you MUST split the integral at each breakpoint and integrate each piece separately.

Key integrals: ∫x² dx = x³/3 | ∫(2−x) dx = 2x − x²/2 | ∫x(2−x) dx = x² − x³/3
